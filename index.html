Git & GitHub Architect Hub

The exhaustive tracking protocol reference. From initial configuration to rewriting history trees.

Pro Aliases (Shortcuts) Pro

git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
Sets up massive keystroke savers. You can now just type git st or git co.
git config --global alias.unstage 'reset HEAD --'
Creates git unstage [file] to quickly remove a file from the current commit queue.
git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"
The ultimate visual tree shortcut. Type git lg to see a beautiful visual history map.

Stage & Snapshot Noob

git add .
Stages all current changes. The most commonly used command.
git add -p
(Pro Shortcut) Interactive patch mode. Lets you review and stage specific chunks of a file instead of the whole thing.
git commit -m "Refactor to POJO-based architecture"
Records file snapshots permanently in version history.
git commit --amend --no-edit
Adds staged files to the last commit without changing its message. A massive lifesaver.

Branch & Merge Mid

git switch -c feature/flutter-glass-ui
The modern, safer alternative to checkout -b. Creates and switches to a new branch.
git branch -D [branch-name]
Force deletes a branch even if it has unmerged changes. Use with caution.
git merge [branch] --squash
Takes all commits from the target branch and puts them into a single, clean commit on your current branch.

Rebase & Cherry-Pick Pro

git pull --rebase origin main
Fetches remote changes and rebases your local commits on top. Keeps history perfectly linear.
git rebase -i HEAD~4
Interactive mode. Allows you to squash, rename, drop, or reorder your last 4 commits.
git cherry-pick [commit-hash] -n
Pulls a specific commit from anywhere, but the -n (no-commit) flag stages it so you can edit before committing.

Stash & Worktrees Pro

git stash push -m "WIP offline-first SQLite schema"
Temporarily stores modified files with a descriptive label so you don't lose track of it in git stash list.
git worktree add ../dev-club-projects main
The ultimate architect trick. Checks out a branch in a totally separate physical folder so you can work on two branches simultaneously without stashing.
git worktree remove ../dev-club-projects
Safely cleans up the linked worktree directory.

Undo & Data Recovery Pro

git reset --soft HEAD~1
Undoes the last commit, but keeps all the file changes staged and ready.
git reset --hard origin/main
Destructive! Completely forces your local branch to perfectly match the remote branch, discarding local changes.
git reflog
The ultimate safety net. Logs every time HEAD moves. Use this to recover "deleted" commits or branches.
git clean -fd
Forcefully deletes untracked files and directories from your workspace.

Debug & Inspect Mid

git bisect start
git bisect bad
git bisect good [working-hash]
Automates finding a bug. It uses binary search to pinpoint exactly which commit broke the code.
git diff --staged
Shows what you are about to commit (differences between index and last commit).
git blame [file]
Shows who modified each line of a file, in which commit, and when.

GitHub CLI (`gh`) Mid

gh repo create PrivLedger --public --clone
Creates a new public repo directly on GitHub and clones it locally in one step.
gh pr create --fill
Instantly opens a Pull Request using the current branch and commit messages for the PR details.
gh repo view --web
Opens the current repository in your default web browser.
